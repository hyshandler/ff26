# Games-Played Estimate is a heuristic in v1, not a trained submodel

A trained submodel (age, injury history, snap trend → predicted games played) was the obvious-seeming choice, since we're treating everything else as a model. Research changed our mind: PFF's own team-level analysis found only weak (~0.33) year-over-year correlation in games lost, and a clinical study found no elevated pre-injury workload pattern — no primary source demonstrates real player-level predictive signal for availability. Building a submodel now would spend effort on a plausibly-intractable problem instead of validating the core PPG model. v1 uses an age/position-conditioned base rate plus a recent-major-injury flag; a real submodel is v2, contingent on a cheap correlation check against nflverse's `load_injuries()` data first. See `docs/research/fantasy-football-projection-modeling.md`.

**Outcome (issue #9)**: the correlation check ran (RB player-seasons 2009-2023, n=2073): injury-report volume vs. same-season actual games played gave a weak *positive* correlation (Pearson r=0.28, Spearman ρ=0.29), most likely an exposure confound (more games played → more chances to appear on a report) rather than real predictive signal. This doesn't cleanly trigger a v2 submodel, and doesn't cleanly reaffirm the null either — see `docs/research/injuries-games-played-correlation.md`. v1's heuristic is unchanged; if games-played modeling is revisited, a year-over-year, status-filtered design (closer to the PFF study cited above) is the recommended follow-up check, not this one repeated.
